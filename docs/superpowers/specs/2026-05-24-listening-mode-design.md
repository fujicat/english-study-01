# リスニングモード追加 設計書

作成日: 2026-05-24
対象ファイル: `index.html`

## 概要

英語学習アプリ（Homestay English）に新しい学習モード「リスニングモード」を追加する。

学習フロー:
1. 問題表示時に英語音声を自動で1回再生（テキストは非表示）
2. ユーザーは何度でも音声を再生できる
3. 「ANSWER」ボタンで英文を表示（答え合わせ）

## 背景

既存の4モード（英→日 / 日→英 / 英→🔊 / 日→🔊）はすべて、問題として「テキスト」が画面に表示される。リスニング学習に特化した「音声だけを聞いて英文を頭の中で再現する」モードが欠けているため、これを追加する。

## アーキテクチャ

- 変更箇所は `index.html` のみ（単一ファイル構成を維持）
- 新規ファイル・新規依存ライブラリは追加しない
- 既存の4モードの挙動は一切変更しない（純粋な追加）
- 既存の `MODES` オブジェクト、`speak()` 関数、Next/Prev/Shuffle/Reset 等のロジックを最大限再利用する

変更ポイントは以下3箇所のみ:
1. `MODES` 定義の拡張
2. 問題画面のレンダリング分岐
3. 自動再生 useEffect の追加

## データ構造

`MODES` を以下のように拡張する。

```js
const MODES = {
  en2ja:     { label: '英 → 日',  q: 'en',    a: 'ja', voice: false, autoPlay: false },
  ja2en:     { label: '日 → 英',  q: 'ja',    a: 'en', voice: false, autoPlay: false },
  en2voice:  { label: '英 → 🔊', q: 'en',    a: 'en', voice: true,  autoPlay: false },
  ja2voice:  { label: '日 → 🔊', q: 'ja',    a: 'en', voice: true,  autoPlay: false },
  listening: { label: '🔊 → 英', q: 'audio', a: 'en', voice: true,  autoPlay: true  },
};
```

### フィールドの意味

| フィールド | 値 | 意味 |
|---|---|---|
| `q` | `'audio'` | 新規。問題は音声のみ。テキストは非表示 |
| `autoPlay` | `true` | 新規。問題切替時に1回だけ自動再生 |
| 既存4モード | `autoPlay: false` | 既存挙動を明示（追記のみ、動作変更なし） |

## 画面仕様

### 問題画面（`showAnswer === false`）

リスニングモードのとき:

- 画面中央に大きな🔊アイコン（既存の `Volume2` SVGコンポーネントを `size=96` 前後で再利用）
- その下に「REPLAY」ボタン
- 英文・日本語訳は一切表示しない
- 文字数や単語数のヒントも表示しない

その他のモード: 既存通り `questionText` を表示。

### 答え画面（`showAnswer === true`）

リスニングモード（`mode.a === 'en'` かつ `mode.voice === true`）は、既存の `en2voice` モードと同じ答え画面を再利用する。具体的には:
- 上部に大きく英文を表示
- 横に🔊再生ボタン（既存「PRONUNCIATION」セクション）

新しいレンダリング分岐は不要。既存実装をそのまま使う。

### タブ

既存タブ列の末尾に「🔊 → 英」を追加する。タブ実装は `Object.entries(MODES)` ベースなので、`MODES` への追加だけでタブが自動生成される。

## 自動再生ロジック

新しい `useEffect` を追加する。

```js
useEffect(() => {
  if (!showAnswer && mode.autoPlay && current) {
    speak(current.en);
  }
}, [pos, modeId, showAnswer]);
```

### 動作タイミング

| タイミング | 動作 |
|---|---|
| リスニングタブを選択 | 現在の問題を1回自動再生 |
| Next / Prev | 新しい問題を1回自動再生 |
| Shuffle / Reset | 新しい問題を1回自動再生 |
| REPLAY ボタン押下 | 既存 `speak()` で再生（何度でも可） |
| ANSWER 押下 | 既存ロジックで再生（変更なし） |
| 答え画面から次の問題へ | 新しい問題を1回自動再生 |

### ブラウザの自動再生制限

タブ切替・Nextボタン等はユーザー操作とみなされるため、`speechSynthesis` は問題なく動作する。デフォルトモードは `en2ja` のままなので、初回ロード時の問題はない。

## エラーハンドリング

既存の `speak()` 関数が `window.speechSynthesis` の存在チェック等を行っているため、新しいエラーハンドリングは不要。リスニングモードでも同じ `speak()` を呼ぶ。

## テストと検証

このプロジェクトにはテストフレームワークがないため、手動ブラウザ検証で完了とする。

### 検証項目

1. リスニングタブを選択 → 英語音声が自動で1回流れる
2. 問題画面に大きな🔊アイコンとREPLAYボタンのみが表示される（英文・日本語は非表示）
3. REPLAYボタンを複数回押す → 毎回再生される
4. ANSWERボタン → 英文が表示される、音声ボタンも表示される
5. Next/Prev で問題切替 → 新しい問題が自動再生される
6. 既存4タブ（英→日 / 日→英 / 英→🔊 / 日→🔊）の挙動が一切変わっていない
7. Shuffle/Reset 動作確認

### 検証方法

`open index.html` でブラウザを開き、上記7項目を目視確認。

## 影響範囲

- 既存4モードの動作: **変更なし**
- 既存のlocalStorage（`modeId` 永続化があれば）: 既存値はそのまま有効。新規 `listening` モードは追加されるだけ
- パフォーマンス: 影響なし（`useEffect` の依存配列は既存と同じ変数のみ）

## YAGNI（やらないこと）

- 文字数ヒントや単語数ヒントは表示しない
- 答え合わせ時の日本語訳表示は行わない（リスニングに集中）
- 再生速度の変更機能は追加しない（既存 `speak()` の挙動を流用）
- 自動再生回数の設定は持たない（常に1回）
